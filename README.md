# 🚀 QuickSend 使用指南

## 👥 面向人群
- 适合普通用户，无需懂技术或命令行
- 一台电脑作为“主机”，另一台电脑/手机在同一 Wi‑Fi 下即可使用

## 🧭 你能用它做什么
- 🗂️ 在电脑与手机/另一台电脑之间快速传文件
- 📝 分享一段文字或链接，支持密码
- 🔐 给文件/文字加密码，保护隐私
- 🧩 用分组整理内容，支持搜索与预览

## 🧰 准备工作
- 主机电脑（Windows）上安装或运行 QuickSend
- 手机/另一台电脑与主机在同一 Wi‑Fi 或热点下

## 📦 安装与启动
- 安装包方式
  - 双击安装包，按提示完成安装
  - 从桌面或开始菜单打开 QuickSend
- 便携运行（免安装/仓库内运行）
  - 在项目目录双击 `start_quicksend.bat` 或运行 `python quicksend\app.py`
- 启动后会弹出应用窗口或自动打开浏览器页面

## 🏠 主界面速览
- 顶部显示“我的地址”和二维码
  - 手机或另一台电脑用浏览器打开该地址，或扫码进入
- 两个主要标签页
  - 文件：上传/下载/预览/分组/密码
  - 文本：新建/复制/密码/删除
- 右上角常用入口
  - 设置：修改保存位置、切换传输模式
  - 登录/注册：创建自己的账号，用于标记“上传者”和授权删除/改密
  - 打开目录：在系统文件夹里查看/定位已上传内容

## 🌐 让其他设备连接
- 确保在同一 Wi‑Fi / 热点
- 在主机页面复制“我的地址”（形如 `http://192.168.x.x:5000`）或扫码
- 在手机/另一台电脑的浏览器中打开即可
- 打不开时
  - 确认都连在同一网络（不是蜂窝/数据流量）
  - 主机防火墙弹窗允许访问，或稍等自动放行后重试
  - 如果用 NAS/Docker 部署，确认端口映射和地址正确

## 🗂️ 文件分享
- 上传文件
  - 在“文件”页点击“上传”或把文件拖进去
  - 可选：选择分组、设置密码、填写“上传者”（建议填你的昵称）
- 下载与预览
  - 所有连接到页面的设备都能看到列表，点击即可下载
  - 图片/音频/CSV/XLSX/DOCX 等可在页面内预览（部分格式需主机安装预览组件）
- 分享给别人
  - 复制页面地址后发给对方（同一网络）
  - 在文件卡片里复制直链或让对方扫码访问
- 设置/取消密码
  - 在文件的“更多/密码”入口中设置或清除
  - 只有“上传者本人”或“主机”可以改密或清除
- 删除与移动分组
  - 删除：只有上传者或主机可删除
  - 移动：把文件拖到分组或在“移动到分组”里选择目标分组
- 打开保存位置（主机）
  - 点击“打开目录”或“定位文件”，系统会弹出该文件夹/文件位置

## 📝 文本分享
- 新建文本
  - 切到“文本”页，点击“新建”输入内容，必要时设置密码
- 分享与复制
  - 创建后生成一条列表项，点击即可复制内容或复制访问链接
- 密码与删除
  - 文本同样支持设置/取消密码；只有上传者或主机可删除

## 📚 分组与搜索
- 创建分组：输入名字即创建，支持多层级
- 重命名/移动分组：在“更多”操作里完成；拖动分组可调整层级
- 隐藏分组（仅主机）：主机可将分组设为“隐藏”，远端设备看不到其内容
- 搜索：顶部搜索框支持模糊匹配，快速定位文件/文本

## ⚙️ 设置与模式
- 修改保存位置：在“设置”中选择新的保存文件夹，之后所有上传都会存到该位置
- 传输模式
  - 分享模式（默认）：远端可浏览主机已有内容
  - 单向模式：远端只能上传、不能浏览（适合收作业/收素材）
- 登录/注册
  - 首次可注册用户名和密码；登录后你的上传会标记为“你”，便于删除或改密

## 🔐 权限与安全
- 仅本机可执行：修改配置、隐藏分组、打开目录、退出程序
- 上传者或本机可执行：删除、设置/取消密码、移动文件分组
- 给敏感文件设置密码；必要时配合“隐藏分组”使用

## 🆘 常见问题
- 对方打不开链接
  - 检查是否在同一网络；换浏览器试试；主机稍等防火墙自动放行或手动允许
- 上传很慢或失败
  - 尽量在信号好的 Wi‑Fi；大文件建议有线连接；必要时分批上传
- 手机不能在线预览某些格式
  - 让手机先下载后用本地 App 打开；或在主机安装 Office/预览组件以改善在线预览
- 忘记密码
  - 只有主机或上传者能清除；请联系主机
- 如何退出
  - 关闭应用窗口即可；若在浏览器打开，关闭页面后主机仍在运行，回到主机窗口退出或在“设置/更多”里选择退出

## 💡 小贴士
- 用二维码让别人更快进入你的页面
- 给重要文件加密码并放在“隐藏分组”（仅主机可见）
- 设置常用保存文件夹，避免文件分散
- 搜索支持模糊匹配，文件名记不清也能快速定位

## 📎 附：离线安装

> 这是一个无需联网、无需构建代码即可部署的 QuickSend 版本。

### 📂 文件清单
请确保你已经收到了以下文件：
1. `quicksend_v6.2_arm64.tar`（镜像文件）
2. `docker-compose.yml`（配置文件）

### 🚀 安装步骤
**第一步：上传文件**  
在你的 NAS 或服务器上创建一个新文件夹（例如 `quicksend`），将上述两个文件上传到该文件夹中。

**第二步：导入镜像**  
打开终端（SSH 或 NAS 的 Docker 终端），进入该文件夹，运行：
```bash
sudo docker load -i quicksend_v6.2_arm64.tar
```
执行成功后，会看到类似 `Loaded image: quicksend-quicksend:latest` 提示。

**第三步：启动服务**  
继续在终端中运行：
```bash
sudo docker compose up -d
```
如果提示 `docker compose` 不存在，请尝试 `sudo docker-compose up -d`。

**第四步：开始使用**  
打开浏览器访问：`http://<你的NAS_IP>:8080`

### ❓ 常见问题
**1. 页面显示的 IP 地址不对（如 172.x.x.x）**  
Docker 网络隔离导致。打开 `docker-compose.yml`，找到 `environment` 下的 `HOST_IP`，取消注释并改为你 NAS 的真实局域网 IP：
```yaml
environment:
  - HOST_IP=192.168.1.10  # 修改为你自己的 IP
```
保存后运行 `docker compose up -d` 重启服务。

**2. 如何修改端口？**  
修改 `docker-compose.yml` 的 `ports`：
```yaml
ports:
  - "8888:5000"  # 左边是你要访问的端口，右边 5000 不要动
```

**3. 文件保存在哪里？**  
默认情况下，上传的文件会保存在当前目录下的 `uploads` 文件夹中（或你在“设置”里更改后的目录）。
